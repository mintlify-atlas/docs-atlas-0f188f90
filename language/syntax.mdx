---
title: Syntax overview
description: Neutronium's C-like syntax and grammar rules
---

Neutronium uses a clean, C-like syntax with modern ergonomics. The language emphasizes readability with explicit block delimiters and minimal punctuation.

## Comments

Comments start with `#` and continue to the end of the line:

```neutronium
# This is a comment
let x = 5;  # Inline comment
```

## Program Structure

A Neutronium program consists of external function declarations followed by function definitions:

```neutronium
# External declarations
extern fn print_num(num: int);
extern fn abs(val: int) -> int;

# Function definitions
fn main(): {
    let x = 42;
    print_num(x);
}
```

## Statements

Neutronium supports the following statement types:

- **Block statements**: `{ ... }`
- **Variable definitions**: `let x = 5;`
- **Assignments**: `x = 10;` or compound `x += 5;`
- **Control flow**: `if`, `while`, `break`, `continue`, `return`, `exit`
- **Expression statements**: `function_call();`
- **Comments**: `# comment text`

All statements except blocks and control flow keywords must end with a semicolon (`;`).

## Expressions

Neutronium supports rich expressions with standard operator precedence:

```neutronium
let result = 1 - 42 * (7 / 3) + 5;
let comparison = x > 10 && y < 20;
let value = array[2] * 10;
```

### Operator Precedence

From highest to lowest:

1. **Primary**: literals, identifiers, function calls, parentheses `()`
2. **Postfix**: array indexing `[]`
3. **Unary**: `-`, `+`, `!`
4. **Multiplicative**: `*`, `/`
5. **Additive**: `+`, `-`
6. **Comparison**: `==`, `!=`, `<`, `<=`, `>`, `>=`
7. **Logical AND**: `&&`
8. **Logical OR**: `||`

## Block Syntax

Control flow statements use a colon (`:`) followed by a block:

```neutronium
if x > 0: {
    print_num(x);
}

while counter < 10: {
    counter += 1;
}

fn example(): {
    # function body
}
```

<Note>
The colon (`:`) before blocks is mandatory in Neutronium. This distinguishes it from C-style languages that use braces alone.
</Note>

## Formal Grammar

<Accordion title="Complete EBNF Grammar">

```ebnf
compilation-unit ::= { external-function-declaration } { function-definition } EOF

function-signature ::= identifier '(' parameter-list ')' [ '->' type-specifier ]

external-function-declaration ::= 'extern' 'fn' function-signature ';'

function-definition ::= [ 'export' ] 'fn' function-signature ':' block-statement

statement ::= block-statement
            | variable-definition
            | assignment
            | if-statement
            | while-statement
            | break-statement
            | continue-statement
            | return-statement
            | exit-statement
            | expression-statement
            | comment

block-statement ::= '{' { statement } '}'

type-specifier ::= 'int' | 'int8' | 'int16' | 'int32' | 'int64'
                 | 'bool'
                 | array-type
                 
array-type ::= '[' type-specifier ';' integer-literal ']'

variable-definition ::= 'let' [ 'mut' ] identifier [ ':' type-specifier ] '=' expression ';'

assignment ::= expression ('=' | '+=' | '-=' | '*=' | '/=') expression ';'

parameter-list ::= [ parameter-declaration { ',' parameter-declaration } ]

parameter-declaration ::= [ 'mut' ] identifier ':' type-specifier

if-statement ::= 'if' expression ':' block-statement { elif-clause } [ else-clause ]

elif-clause ::= 'elif' expression ':' block-statement

else-clause ::= 'else' ':' block-statement

while-statement ::= 'while' expression ':' block-statement

break-statement ::= 'break' ';'

continue-statement ::= 'continue' ';'

return-statement ::= 'return' ';'
                   | 'return' expression ';'

exit-statement ::= 'exit' expression ';'

expression-statement ::= expression ';'

comment ::= '#' { any-character-except-newline }

expression ::= logical-expression

logical-expression ::= comparison-expression
                     | or-chain-expression
                     | and-chain-expression

or-chain-expression ::= or-chain-expression '||' comparison-expression

and-chain-expression ::= and-chain-expression '&&' comparison-expression

comparison-expression ::= additive-expression
                        | additive-expression ('==' | '!=' | '<' | '<=' | '>' | '>=') additive-expression

additive-expression ::= multiplicative-expression
                      | additive-expression ('+' | '-') multiplicative-expression

multiplicative-expression ::= unary-expression
                            | multiplicative-expression ('*' | '/') unary-expression
                            
unary-expression ::= postfix-expression
                    | unary-op postfix-expression
                    
unary-op ::= '-' | '+' | '!'

postfix-expression ::= primary-expression
                     | postfix-expression '[' expression ']'
                                        
primary-expression ::= literal
                     | identifier
                     | function-call
                     | '(' expression ')'

literal ::= integer-literal | boolean-literal | array-literal | repeat-array-literal

array-literal ::= '[' expression-list ']'

repeat-array-literal ::= '[' expression ';' integer-literal ']' 

function-call ::= identifier '(' expression-list ')'

expression-list ::= [ expression { ',' expression } ]

identifier ::= letter { letter | digit | '_' }

integer-literal ::= digit { digit }

boolean-literal ::= 'true' | 'false'

letter ::= 'a'..'z' | 'A'..'Z'

digit ::= '0'..'9'
```

</Accordion>

## Real-World Example

Here's a complete example from the Neutronium standard library:

```neutronium
# Returns the absolute value of a
export fn abs(val: int) -> int: {
    if val < 0: {
        return -val;
    }
    return val;
}

# Returns a^b (a to the power of b)
export fn pow(a: int, b: int) -> int: {
    let mut result = 1;
    let mut i = 0;
    while i < b: {
        result = result * a;
        i = i + 1;
    }
    return result;
}
```
