## Your First Neutronium Program

Let's create a simple program to get started with Neutronium. This guide will walk you through writing, compiling, and running your first program.

<Steps>
  <Step title="Create a Source File">
    Create a new directory for your Neutronium programs and create a file called `hello.nt`:

    ```bash
    mkdir -p scripts
    cd scripts
    ```

    <Note>
      The `scripts/` directory is ignored by version control, making it a good place to experiment with your own code.
    </Note>

    Now create `hello.nt` with the following content:

    ```neutronium hello.nt
    # External I/O functions from the runtime
    extern fn print_c(char: int);
    extern fn print_num(num: int);

    fn main(): {
        # Print "Hello" character by character
        print_c(72);   # H
        print_c(101);  # e
        print_c(108);  # l
        print_c(108);  # l
        print_c(111);  # o
        print_c(10);   # newline

        # Print a number
        print_num(42);
        print_c(10);   # newline

        return;
    }
    ```

    This program demonstrates:
    - External function declarations (`extern fn`)
    - The `main()` function entry point
    - Function calls
    - Comments using `#`
  </Step>

  <Step title="Compile the Program">
    From the repository root directory, compile your program using the `compile.sh` script:

    ```bash
    cd ..
    ./compile.sh scripts/hello.nt
    ```

    The compiler will:
    1. Load and lex the source file
    2. Parse it into an AST
    3. Perform semantic analysis
    4. Generate x86-64 assembly code
    5. Assemble and link with the runtime library

    The compiled executable is written to `neutro/out`.
  </Step>

  <Step title="Run the Program">
    You can either run the executable directly:

    ```bash
    ./neutro/out
    ```

    Or use the convenient `run.sh` script which compiles and runs in one step:

    ```bash
    ./run.sh scripts/hello.nt
    ```

    You should see:
    ```
    Hello
    42
    ```
  </Step>
</Steps>

## A More Complex Example

Here's a program that demonstrates more Neutronium features:

<CodeGroup>
```neutronium fibonacci.nt
# External functions
extern fn print_num(num: int);
extern fn print_c(char: int);

# External math function from runtime
extern fn max(a: int, b: int) -> int;

# Calculate nth Fibonacci number
fn fibonacci(n: int) -> int: {
    if n <= 1: {
        return n;
    }
    return fibonacci(n - 1) + fibonacci(n - 2);
}

# Print the first n Fibonacci numbers
fn main(): {
    let mut i = 0;
    let count = 10;
    
    while i < count: {
        let fib = fibonacci(i);
        print_num(fib);
        print_c(10);  # newline
        i += 1;
    }
    
    return;
}
```

```bash Compile and Run
./run.sh scripts/fibonacci.nt
```
</CodeGroup>

This example demonstrates:
- **Variables**: `let mut` for mutable variables, `let` for immutable
- **Functions**: Function definitions with parameters and return types
- **Control Flow**: `if`/`elif`/`else` conditionals and `while` loops
- **Recursion**: The `fibonacci` function calls itself
- **Operators**: Arithmetic (`+`, `-`), comparison (`<`, `<=`), compound assignment (`+=`)

## Language Features

Neutronium supports:

- **Types**: `int`, `int32`, `bool`, arrays (`[int32; 5]`)
- **Control Flow**: `if`/`elif`/`else`, `while`, `break`, `continue`, `return`
- **Functions**: Named functions with parameters and optional return types
- **Variables**: Mutable (`let mut`) and immutable (`let`) bindings
- **Arrays**: Fixed-size arrays with indexing
- **Export/Extern**: Module system with `export` and `extern` keywords

## Compiler Options

You can inspect intermediate compilation stages:

```bash
# View the Abstract Syntax Tree
./compile.sh scripts/hello.nt --log-ast

# View generated assembly
./compile.sh scripts/hello.nt --log-assembly

# Stop after parsing (don't generate code)
./compile.sh scripts/hello.nt --only-parse

# Enable all debug logging
./compile.sh scripts/hello.nt --debug
```

## Output Location

As of October 2025, the compiler always writes the generated executable to `neutro/out`. All intermediate files (assembly `.asm`, object `.o`) are also placed in the `neutro/` directory.
